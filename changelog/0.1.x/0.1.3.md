# 0.1.3 — 2026-06-30

Security patch clearing every outstanding `bun audit` advisory, alongside four create-path hardening fixes so a successful `workflow_create` always round-trips through the same parser and indexer used at startup.

## Fixed

- **`workflow_create` cross-category duplicates** — a second permanent workflow with the same `name@version` under a different category is now rejected with `already_exists` before writing. `writePermanent()` checks the category-independent index key first; previously the duplicate wrote a second file that collapsed to one last-write-wins index entry. ([#7](https://github.com/cyanheads/workflows-mcp-server/issues/7))
- **YAML serialization** — `workflow_create` and `workflow_create_temp` serialize workflows with `yaml.stringify` (the hand-rolled `buildYaml`/`quote`/`yamlValue` helpers are removed), so coercion-prone scalars like `"true"`/`"123"` and special mapping keys such as `"bad: key"` round-trip as strings and valid mappings instead of parsing back as booleans/numbers and dropping out of the index. ([#8](https://github.com/cyanheads/workflows-mcp-server/issues/8))
- **Semver validation** — the version field uses full `semver.valid()` validation via `.refine()` in place of the start-anchored `^\d+\.\d+\.\d+` regex, at both the create inputs and `WorkflowSchema.version`. A suffix like `1.0.0junk` is rejected at create and skipped at index build, so latest-version lookup (`semver.rcompare`) can no longer crash on indexed data. ([#9](https://github.com/cyanheads/workflows-mcp-server/issues/9))
- **`workflow_create` empty-slug categories** — a category that slugifies to an empty string (e.g. `!!!`) is rejected as `invalid_input` instead of writing into the `categories/` root with no subdirectory; mirrors the existing name-slug guard. The default `uncategorized` path (category omitted) is unaffected. ([#10](https://github.com/cyanheads/workflows-mcp-server/issues/10))
