# 0.1.0 — 2026-05-28

## Added

- `workflow_list` — list permanent workflows with optional category/tag filters; `includeTools` flag surfaces `server/tool` pairs across steps
- `workflow_get` — retrieve complete workflow definition by name; semver-aware (latest when version omitted); injects `global_instructions.md` content
- `workflow_create` — write a permanent workflow YAML to `categories/<slugified-category>/`; rejects duplicate `name@version`
- `workflow_create_temp` — write a temporary workflow to `temp/`; indexed but excluded from list results; no conflict check
- `WorkflowIndexService` — in-memory index keyed by `name@version`; YAML validation at load time (lenient — invalid files skipped and logged); filesystem watcher (Node.js `fs.watch` recursive) with configurable debounce
- Semver-aware lookup: highest version returned when `version` omitted on `workflow_get`
- `_index.json` snapshot written to workflows root on every rebuild
- Configurable via `WORKFLOWS_DIR`, `GLOBAL_INSTRUCTIONS_PATH`, and `WATCHER_DEBOUNCE_MS`
- Seed `workflows-yaml/` directory with example workflows and `global_instructions.md`
- STDIO and Streamable HTTP transports via `@cyanheads/mcp-ts-core ^0.9.11`
