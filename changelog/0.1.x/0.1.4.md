# 0.1.4 — 2026-06-30

Two hygiene fixes for a clean first boot — a fresh install pointed at a not-yet-created workflow root, and the bundled seed library.

## Fixed

- **Missing `WORKFLOWS_DIR` on first run** — `WorkflowIndexService.init()` now runs `fs.mkdir(root, { recursive: true })` before the first `rebuild()` and `startWatcher()`, so an install whose configured root does not exist yet lands its `_index.json` snapshot and keeps the filesystem watcher attached, instead of both ENOENT-ing and leaving the server "ready" but silently un-watched. Mirrors the lazy `mkdir` already used by `writePermanent()`/`writeTemp()`. ([#11](https://github.com/cyanheads/workflows-mcp-server/issues/11))
- **Bundled research-visualization workflow category** — the shipped `pubmed-research-with-cosmograph-visualization-workflow.yaml` now declares `category: Research Visualization` (slug `research-visualization`, matching its parent directory), silencing the "missing category field" startup warning and letting category-filtered `workflow_list` match it. ([#12](https://github.com/cyanheads/workflows-mcp-server/issues/12))
