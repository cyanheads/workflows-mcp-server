# 0.1.1 — 2026-05-28

Complete rewrite of `workflows-mcp-server` on `@cyanheads/mcp-ts-core ^0.9.11`. Published as `@cyanheads/workflows-mcp-server` (scoped package); the old unscoped `workflows-mcp-server@1.0.5` on npm is unchanged.

## Added

- **`workflow_list`** — list permanent workflows with optional `category` and `tags` filters (AND match); `includeTools` flag surfaces `server/tool` pairs across steps
- **`workflow_get`** — retrieve complete workflow definition by name; semver-aware (highest version when `version` omitted, exact match when specified); injects `global_instructions.md` content into every response
- **`workflow_create`** — write a permanent workflow YAML to `categories/<slugified-category>/`; rejects duplicate `name@version` to prevent silent overwrites
- **`workflow_create_temp`** — write a temporary workflow to `temp/`; indexed on the fly but excluded from `workflow_list` results; no conflict check
- **`WorkflowIndexService`** — in-memory index keyed by `name@version`; YAML validation at load (lenient — invalid files skipped and logged); native Node.js `fs.watch` recursive watcher with configurable debounce; `_index.json` snapshot written on every rebuild
- Seed `workflows-yaml/` with example workflows and `global_instructions.md`
- STDIO and Streamable HTTP transports
- Typed error contracts: `not_found`, `version_not_found`, `already_exists`, `index_unavailable`
- 71 tests across 5 files (tools + service)

## Breaking

- **Package name changed** from `workflows-mcp-server` to `@cyanheads/workflows-mcp-server` — update `package.json` and MCP config to use the scoped name
- Tool names, YAML schema, and env vars are new (`WORKFLOWS_DIR`, `GLOBAL_INSTRUCTIONS_PATH`, `WATCHER_DEBOUNCE_MS`); v1.x config does not carry over
